Case study for myFlix App

Overview

MyFlix web app is developed using the MERN stack that allows users to sign up, log in and get information about movies, genres, and directors. It also allows user to update their account information, add movies in their list of favorites and remove movies from their list of favorites.

Purpose & Context

Create an interactive and enjoyable application for movie lovers like myself who enjoy reading information about different movies. MyFlix was a personal project I built as part of my web development course at CareerFoundry to grasp the concept of the MERN stack.

Objective

The main objective of the project was to have a fully functioning full-stack web app where I can showcase my skills and add it to my portfolio. The problem I wanted to solve is to build a complete app from scratch (both server-side and client-side).

Approach

I used MERN stack to develop this app.

  • First Create a database to save movie details, user information and API endpoints for accessing data
  • Develop responsive client side with necessary features so that user can access the app content
  • Host this app on github

Server-side

  • Created a Rest-full API with Node and route http requests using Express.
  • Also built a NoSQL database using MongoDB.
  • Modeled the business logic using Mongoose.
  • Implemented authentication and authorization using basic HTTP authentication and JWT (token-based) authentication.
  • API endpoints were tested via Postman.
  • This API is hosted on Heroku.

Client-side

  • Used React Framework to create this Single page application
  • Used React-Router-Dom to navigate between views and share URLs
  • Used Bootstrap as a UI library for styling and responsiveness
  • Used Redux and React-Redux to manage the app's state and data flow by implementing the flux design pattern
  • This app is hosted on Netlify

Challenges

This was my first ever MERN project. As much as I have enjoyed building this app, I also faced a lot of challenges. Building server-side was a smooth process but not the client-side with React. As a beginner, it took me a while to understand the class component, function component and react, react-rdux . However, after a lot of research and with the help of my mentor I understood the concept.
Problem: When Updating the profile, if the user doesn't update the password field and hit submit, my application was submitting the user hashed password (which is retrieved to show user info) to the database. As a result, when a user tries to log in, it was giving the user a warning "invalid password". Even though the user is giving the right password, but in the database password is different.
Solution: After a lot of troubleshooting, I figured out the problem. With the help of my mentor and browsing other renowned sites like Facebook, Twitter, Instagram, I realized my user info section supposed to be without showing the password. After removing the password field and only giving the user to update the password solved the problem.

Duration

To build the server-side of this application I finished it before the allocated time but building the client-side, took me extra two weeks to complete. The reason behind the extra time is that I needed time to familiarize myself with React and as well as React-Redux.
